Cs 2005 Project Team K

User story

After logging in,the user picks their desired subscribed course which reveals the required chapters. Upon clicking the chapter that the user wants, it displays the lecture material on one side and external links to all the required readings for the specific chapter on the other side (they may be ranked in order of relevance). Users can edit that part of the website by adding more links or information, and by voting on which required readings are the most important.

Use case

Title:User accessing an external reading and posting an interesting article about software engineering cs2005

Main success scenario:
  1. The user loggs in.
  2. The user picks the desired subject, in this case Cs2005.
  3. The user is then prompted with the chapters and sub chapters.
  4. The user picks the desired lecture.
  5. The screen splits vertically into two with the right half being the lecture material and the left the external readings ranked in order of importance.
  6. The user then examines the lecture material and clicks the desired external reading.
  7. The user stumbles upon an interesting reference in the external reading and decides to share it.
  8. The user posts the link in a forum-based interface on the same left side where all the external documents are.
  9. Other users get to rank the readings in order of importance with the Instructors.
  10. The posts added by all users are to be verified by an admin.
Extensions (some scenario variations):
  • User enters wrong log in credentials (error prompts).
  • User does not find the right course in the outline and needs a log out button.
  • The user decides not to post a link or upvote an already posted link(no contribution effect).
  • The user decides to post a malicious link.
  • The user desices to edit their profile instead of accessing the courses and their material
  • The application crashed due to unexpected load or error.
